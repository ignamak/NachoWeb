DeckDungeon

2020 - Unity / C#

The project I am nowadays working on is DeckDungeon. The name is temporary and stands for its main mechanics, Deck building and Dungeon crawling. The core of the project is randomness and short sessions of gameplay. For now it features random dungeon generation and a random card generator. Doing this project I have learned about random dungeon procedures and how to use Unity's Scriptable Objects to build better game system architectures. This is still a work in progress.

Blackstorm

2019 - Unreal / Blueprints

Done in the context of my Master's Degree in Game Design, Blackstorm is the biggest and most ambitious project I've worked on. The game is a Battle Royale focused on cooperative teamplay, features unique mechanics like multiple gravities in an online game and three axis space navigation. Being part of an interdisciplinary 18 people team taught me how to work in a bigger project where many things are out of your reach. My role within the team was technical design, write documentation, design systems and mechanics and prototype them. We are aiming to launch the game for free in late 2020.

Final Degree Project

2017 - Unity / C#

As a final Bachelor Videogames Design and Development degree project I made a Guitar Hero clone with a twist. It could be played remotely with android devices, it could be played by two people, and it made the levels on the fly with the only input of a MIDI file. During this project I learnt a lot about file formats, parsing, and networks.

The Enigma from Another World

2016 - Unity / C#

A simple IA project where bots were able to move using different patterns like wander, chase or flock. Using graph theory and BFS algorithm I made bots able to remember paths to objects in case of need (go for a healthpack when they lose health).

StealthBot

2015 - Processing / SimpleOpenni & Arduino / Kinect 360

My university brought a guest lecturer and after a 2 hours of workshop we were set to do a 48 hours game jam in which we had to make a game using kinect 360. I thought that it would be fun to mix another passion of mine which is robotics with arduino. So we made a game with no screen, two players and a robot controlled by their bodies. The goal was to complete a circuit in the less amount of time, each player had the control over one wheel, so they had to work as one to take turns and move straight. Turned out to be very fun and had a nice time doing it.
